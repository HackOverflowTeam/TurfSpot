üé® Button Reference

This is how the buttons should look:

‚úÖ Visual Testing Checklist

  • Logo never overlaps with buttons
  • Login and Register buttons are centered horizontally
  • Buttons have consistent spacing (gap visible between them)
  • Buttons never touch the screen edges
  • Button text never wraps to second line
  • Icons inside buttons are always visible
  • Icons maintain proper size (not squished)
  • Hamburger menu aligns perfectly to the right
  • Hamburger doesn't shift logo position
  • No horizontal scrollbar appears on any screen size
  • All elements fit on one line
  • Text remains readable on smallest screen (320px)

üñ±Ô∏è Interaction Testing Checklist

  • Login button opens login modal
  • Register button opens register modal
  • Hamburger menu opens mobile menu
  • Mobile menu slides in smoothly
  • Clicking outside closes mobile menu
  • All buttons have hover effects (desktop)
  • Touch feedback works on mobile
  • User dropdown appears after login
  • Navbar stays fixed at top while scrolling
  • Smooth transitions on all interactions

üìè Spacing Verification

Check These Spacing Rules:

  1. Container Padding: Never less than 0.4rem on smallest screens
  2. Button Gap: Always visible space between Login and Register
  3. Icon Gap: Space between icon and text in buttons
  4. Hamburger Spacing: 12px ‚Üí 8px ‚Üí 6px as screen shrinks
  5. Edge Clearance: Minimum 0.4rem from screen edges

üêõ Common Issues to Check

  • Buttons squished or text cut off
  • Horizontal scrollbar appearing
  • Logo overlapping buttons
  • Hamburger shifting logo position
  • Buttons touching screen edges
  • Text wrapping inside buttons
  • Icons shrinking or disappearing
  • Uneven spacing between elements
  • Navbar height changing unexpectedly
  • Elements jumping during resize
